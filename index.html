Explain more about this Devil-compression.



works by creating every possible file of a given size.
As it does so, it checks each to see if it's SHA matches the original.
When a SHA match is found, the file is then checked against the to see if it's a valid file of the original type.
For example, there will be many SHA matches for an audio file, but very few will be valid MP3s.
To further narrow things down, a perceptual hash can be used at this point.


Of the remaining candidates, we can either check each, or have the original compressor check each, and write down which iteration is correct.
